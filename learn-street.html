SVG graphics

SVG Studio

Scalable Vector Graphics Overview

SVG (Scalable Vector Graphics) is an XML-based language used to describe two-dimensional vector graphics. Unlike standard raster formats (like PNG or JPEG) which map static color values to pixels, SVG stores mathematical definitions of lines, shapes, text, and curves.

✦ Resolution Independent ✦ Compact Files ✦ Scriptable via DOM ✦ Searchable & SEO Friendly
XML Compliance Requirements

Because SVG is an XML dialect, it follows strict syntactic rules. Unlike HTML, all tags must be closed. A tag can be closed either as a self-closing element or with an explicit closing tag:

Self-Closing Shape: <circle cx="50" cy="50" r="40" />
Standard Container: <svg> ... </svg> or <text>Hello</text>
The SVG Grid Geometry

The canvas coordinate system starts at the top-left corner (0, 0). X values increase as you move to the right. Y values increase as you move downward. This is opposite to conventional Cartesian coordinates, which often causes confusion for beginners!

Hover your mouse over the grid to locate coordinates: (X: 0, Y: 0)
Key concept: When drawing shapes, coordinate attributes like x, y, cx, cy, x1, y1 specify offsets relative to this top-left origin.
Predefined SVG Geometries

SVG comes with a variety of basic shape tags. Learn their attributes and meanings:

Shape Tag Attributes Meaning
<rect> x, y, width, height, rx, ry Draws a rectangle. rx, ry define the horizontal and vertical corner rounding.
<circle> cx, cy, r Draws a circle centered at (cx, cy) with radius r.
<ellipse> cx, cy, rx, ry Draws an oval centered at (cx, cy) with X-radius rx and Y-radius ry.
<line> x1, y1, x2, y2 Draws a straight line segment from (x1, y1) to (x2, y2).
<polyline> points Draws multiple connected straight lines (e.g. points="10,20 30,40 50,20").
<polygon> points Like a polyline, but the browser automatically closes the shape by drawing a line back to the starting point.
<path> d The most powerful drawing command. The d (data) attribute defines paths using coordinates and commands: M (move to), L (line to), H (horizontal line), V (vertical line), C (cubic bezier), A (elliptical arc), Z (close path).

Key Regulations

Janet Headquarters

Janet network & Jisc

Janet is the high-speed network for research and education in the UK, provided by Jisc. Compliance with Janet's regulations and policies is critical to ensure network security, data protection, and general integrity.

Regulatory Frameworks
Computer Misuse Act 1990

Addresses unauthorized access and misuse of network resources. This criminalizes hacking, unauthorized data modification, and deliberate malware injection.

GDPR & UK Data Protection Act 2018

Governs the protection and processing of personal data within Janet-connected organizations. Student web development projects must comply with this when capturing or handling user details.

Freedom of Information Act 2000

Governs public access to information held by public institutions, including universities and schools.

NIS Regulations 2018

Focuses on the cyber security of essential services and digital infrastructure, aiming to protect critical network components.

Janet Acceptable Use Policy (AUP)

The AUP defines what constitutes acceptable and unacceptable behavior on the Janet network. All users must ensure:

  • Ethical and lawful use of the network.
  • No deliberate or negligent actions that could harm network security (e.g. running unpatched open-proxy nodes or sharing administrator credentials).
  • No activities that could bring Janet or affiliated institutions into disrepute.
Eligibility Policy

Only authorized organizations can connect to Janet, including:

🎓 Higher Education 🏫 Further Education 🔬 Research Institutions 🏛️ Supporting Public Bodies
Consequences of Violations

Breaching Janet policies can trigger severe consequences, including suspension/termination of network access, institutional disciplinary action, or potential legal prosecution.

CSIRT & Best Practices

Security Operations

Security & Data Protection Responsibilities
  • Access Controls: Only authorized personnel should have access to sensitive data.
  • Encryption & Secure Communication: Required when transmitting sensitive or personal data (e.g. HTTPS, TLS, SSH).
  • Incident Reporting: Any security breaches must be reported to the institution's IT security team, and if necessary, escalated to Janet CSIRT.
What is Janet CSIRT?

The Computer Security Incident Response Team (CSIRT) provides a coordinated response to security incidents across the Janet network. They support institutions in handling cyber threats (such as DDoS attacks, malware outbreaks, and phishing campaigns) and cooperate with national/international security agencies.

Best Practices for Compliance
Best Practice Implementation details
User Awareness Mandatory, recurring cybersecurity training for students and staff.
Strong Authentication Implementing Multi-Factor Authentication (MFA) on all university accounts.
Security Assessments Periodic network and system vulnerability scans, plus penetration testing.
Retention & Disposal Enforcing strict policies for secure data storage and deletion under GDPR guidelines.

Digital Accessibility

Accessibility Labs

Accessibility & Janet Compliance

While accessibility is not explicitly listed in Janet's core regulations, institutions using Janet are legally required to comply with UK digital accessibility legislation.

Legal Frameworks
Public Sector Accessibility Regulations 2018 (UK)

Requires all public sector websites and mobile applications to meet the internationally recognized **WCAG 2.1 AA** standards.

Equality Act 2010

Mandates "reasonable adjustments" to ensure digital resources do not discriminate against individuals with visual, auditory, motor, or cognitive disabilities.

Web Development Guidelines (POUR)

Web resources connected to Janet must adhere to the four accessibility principles:

  • Perceivable: Users must be able to perceive information (e.g. alt text for images).
  • Operable: The interface must be keyboard navigable (avoid mouse-only traps).
  • Understandable: Clear typography, consistent layout, readable text.
  • Robust: HTML code must validate correctly so screen readers can parse it.
